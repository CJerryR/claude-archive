1. 这是什么

Claude Archive 是一个浏览器扩展,在你与 Claude 对话时实时把完整记录保存到你自己的电脑,包括:对话正文、Claude 的思考过程(thinking)、工具调用链、你上传的文件、Claude 生成的文件。它还内置一个仿 Claude 网页版的离线查看器,让你随时翻看已存档的对话。

全程本地:所有数据只写到你电脑的下载目录,不经过任何服务器,扩展也不联网上传。

支持 claude.ai 与镜像/中转站(如 claude.hk.cn),也可自行添加其它中转站域名。

2. 安装

仅支持桌面版 Chrome / Edge(基于 Chromium 的浏览器)。手机版浏览器不支持扩展。
  1. 下载并解压扩展文件夹(claude-archive-extension)。解压后不要删除或移动这个文件夹,扩展运行时一直依赖它。
  2. 打开扩展管理页:地址栏输入 chrome://extensions(Edge 为 edge://extensions)回车。
  3. 打开右上角的 开发者模式 开关。
  4. 点 「加载已解压的扩展程序」,选中刚才解压出的 claude-archive-extension 文件夹。
  5. 扩展出现在列表里即安装成功。建议点扩展栏的拼图图标,把 Claude Archive 固定到工具栏。

3. 第一次使用

  1. 打开 claude.ai(或你的中转站),进入任意一个对话。
  2. 正常聊天即可。扩展会在后台自动捕获;点扩展图标可看到「当前对话」状态变为已捕获 / 完整。
  3. 想立刻存档,点弹窗顶部 「保存当前对话(全量)」。文件会保存到下载目录的 ClaudeArchive/ 下。
  4. 历史旧对话:打开它、等页面加载完,再点「保存当前对话」即可补存。
弹窗顶部就是常用操作:保存当前对话、全部下载、检查完整性、打开查看器、导出索引/日志。设置项在下方可折叠区域里。

4. 本地查看器

点弹窗里的 「打开查看器」,或直接双击扩展文件夹里的 viewer.html。

viewer.html 和 viewer.js 必须放在同一个文件夹里,否则查看器打不开。

查看器提供:仿官网的对话浏览(含思考链时间线、工具调用、文件)、左侧对话列表、分支切换、代码高亮、图片灯箱,以及全局搜索——可按「用户 / AI 回答 / 思考 / 工具·产物」范围、指定对话、时间段搜索关键词,命中高亮、点击跳转、可返回结果继续点。

首次使用点 「选择存档文件夹」,选中你的 ClaudeArchive 文件夹即可载入全部对话。选一次会被记住,之后点侧栏 ↻ 刷新即可,不必每次重选。

5. 多站点 / 中转站

内置支持 claude.ai 和 claude.hk.cn,开箱即用。如果你用过其它中转站(如 cloudlian.cn):

  1. 点扩展图标,展开 「支持的网站 / 中转站」。
  2. 在输入框填入域名(cloudlian.cn 或带 https:// 都行),点 添加。
  3. 浏览器会弹出一次该网站的访问权限请求,点允许。之后在该站点刷新页面,扩展即开始自动存档。
可以同时在线保存多个站点:不同标签页各自独立捕获,A 站和 B 站的对话互不干扰。弹窗里的「当前对话」显示的是你正看着的那个标签页的对话(带站点名)。
权限是逐站点授予的,只对你明确添加的网站生效;没添加的网站扩展不会读取。随时可在列表里点 ✕ 移除并撤销权限。

6. 直写模式与「文件夹即跟踪」

默认情况下文件通过浏览器下载保存。你也可以开启直写模式:在查看器里绑定一次 ClaudeArchive 文件夹,之后所有文件直接写入该文件夹,零下载记录、不在下载栏刷屏。

绑定文件夹后还获得「文件夹即跟踪」:扩展启动时扫描文件夹里每个对话的 conversation.json 自动重建索引——只要文件夹还在,就不必逐个重新打开对话来恢复跟踪,重装扩展也不丢。

浏览器重启后可能要求重新确认一次文件夹权限(这是浏览器安全策略)。失效时扩展自动退回下载模式,点弹窗里的「绑定文件夹 / 恢复索引」即可恢复。

7. 捕获选项详解

选项作用
实时存档(总开关)关闭后扩展不再捕获任何对话。
自动写盘捕获后自动保存到下载目录;关闭则只捕获、需手动点保存。
保存文件下载你上传的与 Claude 生成的文件(图片、文档、压缩包等)。
全量补全抓取回复结束后再抓一次,确保思考链 / 工具调用完整。
保留历史快照每次抓取另存一份带时间戳的 JSON 到 history/,便于查看器多版本合并。
静默下载退回下载模式时,下载完成后从浏览器下载记录移除(文件保留)。直写模式天然无记录。
抓取限速控制文件抓取的平均速率(不限 / 20 / 5 / 1 MB/s),降低带宽与磁盘占用。
调试日志在 Service Worker 控制台打印「捕获→保存」每一步,用于排查问题。

8. 文件如何组织

所有内容保存在下载目录的 ClaudeArchive/ 下,每个对话一个子文件夹:

ClaudeArchive/
  _index.json                         全局对话索引
  <对话名>__<对话uuid前8>/
    conversation.json                 结构化全量数据(查看器读它)
    conversation.md                   Markdown 版,可直接阅读
    history/                          历次抓取的带时间戳快照
    files/<对话uuid前8>/                该对话的所有图片 / 文档 / 产物
图片只能拿到 Claude 服务器提供的 .webp 预览,原始 PNG 无法获取,这是服务器限制,非扩展所能改变。

9. 同名文件的版本(重要)

同一个对话里,不同消息可能产出同名但内容不同的文件(例如先后两条消息各生成一个 report.txt)。扩展用产生该文件的消息 uuid 前 8 位作前缀来区分它们:

files/<对话码>/a1b2c3d4__report.txt     ← 12:00 那条消息产出的版本
files/<对话码>/e5f6a7b8__report.txt     ← 13:00 那条消息产出的版本

在查看器里点击某条消息后面的文件,会精确下载到该消息对应的那个版本,不会串到别的版本。文件名在界面上仍显示为干净的原名(如 report.txt)。

10. 整理与查重

用过多个历史版本后,下载目录里可能积累重复文件或零散布局。提供两种清理方式:

方式一:查重报告(只读)

浏览器打开 tools/cleanup-report.html,选中 ClaudeArchive 文件夹,即可按对话 + 版本归类、用 SHA-256 标出内容重复的文件,并导出 JSON/CSV 清单。它不修改任何文件,只看不动。

方式二:一键整理脚本

运行 tools/organize_archive.py(需 Python 3.8+),自动按内容去重、把文件统一并入 files/<对话码>/ 并加上消息前缀:

# 预览(只打印计划,不动文件)
python organize_archive.py "C:\Users\你\Downloads\ClaudeArchive"

# 确认无误后执行
python organize_archive.py "C:\Users\你\Downloads\ClaudeArchive" --apply
执行前请整目录备份一次。 去重 / 垃圾文件都移入 ClaudeArchive/_trash/(不直接删),你确认后再手动删除。运行后生成 cleanup-report.csv 记录每个文件去向。

11. 隐私与安全

  • 数据不出本机:扩展只把数据写到你的下载目录,不向任何服务器发送。
  • 权限最小化:内置只匹配 Claude 官方与已知镜像;其它中转站需你逐个手动添加并授权,没添加的网站扩展碰不到。
  • 开源可审计:全部源码公开,可自行检查行为。

12. 常见问题

切换到另一个中转站后,弹窗还显示前一个对话?

弹窗「当前对话」显示的是你当前激活标签页的对话。确保你点的是目标站点的标签页;若该站点是新加的中转站,需先在「支持的网站」里添加并授权、刷新页面。两个站点可同时在线、各自独立保存。

能同时保存多个网站吗?

能。每个标签页独立捕获,互不影响。

查看器点了没反应 / 按钮没用?

确认 viewer.html 和 viewer.js 在同一文件夹。若是在扩展页内打开,KaTeX 公式可能因安全策略回退为纯文本(本地双击打开不受影响),其余功能正常。

下载目录里有很多带 (1)(2) 的重复文件?

那是旧版本遗留。新版已修复不再产生;旧的用上面的查重报告或整理脚本清理即可。

找不到图片的原图(PNG)?

Claude 服务器只提供 webp 预览,原始 PNG 无法获取,只能保存为 .webp。

历史旧对话没被存下来?

打开那个对话、等页面加载完成,点「保存当前对话」即可。也可用「全部下载」遍历所有已跟踪对话补存。
