# Release v2.5.0

**发布日期**: 2026-06-11

### 🔧 改进（Changed）
- **[破坏性变更]** 重构产出文件的存储模型：由「每轮一个文件夹」改为「内容指纹版本化」。所有产出文件统一存入 `files/{对话码}/`，按 SHA-256 内容指纹判重，仅在内容真正变化时保存为新版本（`name__v2.ext`、`name__v3.ext`）。
  - 影响范围:输出目录结构变更。旧版本生成的 `files/r{序号}_{时分秒}_{uuid8}/` 轮次目录不再产生；查看器仍向下兼容读取旧目录，历史存档无需迁移即可浏览。

### ⚡ 优化（Improved）
- 大幅消除重复文件:同一文件在多轮出现但内容仅变化数次时，仅保存对应数量的版本，不再产生大量内容相同的副本（实测 11 轮 3 种内容 → 仅 3 个版本）。

### 🐛 修复（Fixed）
- 修复多版本文件管理的根本缺陷：此前按轮次建夹会因服务器仅保留文件最新版本而下载到多份相同文件。
