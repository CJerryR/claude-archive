# ClaudeArchive 整理手册(交给 Claude Code 执行)

> 目的:把 `ClaudeArchive` 里**多个历史版本扩展**产生的、命名/布局各不相同的文件,
> 归并成**一套统一结构**,安全去重、隔离垃圾。
> 适用目录:`<下载目录>/ClaudeArchive`(示例:`C:\Users\17372\Downloads\ClaudeArchive`)。
>
> **执行前先整目录备份一次(整个 ClaudeArchive 复制一份)。** 本手册所有"删除"都先移动到 `_trash/`,绝不直接删。

> ⚠️ **重要纠正(相对旧版手册)**:早期 v2.2-2.4 曾按"轮次"建一堆 `files/r12_131904_xxxx/` 子目录,
> 但 Claude 服务器对同一文件路径**只保留最新版本**,导致那些轮次目录里**装的多半是一模一样的重复文件**。
> 现行 v2.5+ 已改为 **`files/<对话码>/` 单层目录 + 内容指纹版本化(`__vN`)**。
> 本手册的目标是把所有历史布局**统一并入 `files/<对话码>/`**,而不是保留轮次目录。

---

## 0. 一句话目标

每个对话一个顶层目录;该对话**所有产出文件统一放进 `files/<对话uuid前8>/` 这一层**;
**按文件内容(SHA-256)去重**:同名同内容只留一份,内容不同的视为不同版本 -> `<原名>__v2.<ext>`、`__v3`...;
垃圾/重复移到 `_trash/`;会话正本与 `history/` 快照保留不动。

---

## 1. 现状:历史上出现过的全部命名 / 布局

### 1.1 四代文件夹布局(可能同时出现在同一对话里)
| 代 | 布局 | 例子 | 处理 |
|----|------|------|------|
| A 最早(<=v1.8) | `files/<文件>` 平铺 | `files/CLAUDE.txt`、`files/preview.webp` | 合并进 `files/<对话码>/` |
| B 中期(v1.9-2.1) | `files/<对话uuid前8>/<文件>` | `files/85523dd2/F_hero.webp` | **已是目标层**,保留 |
| C 过渡(v2.2-2.4) | `files/r<序号>_<时分秒>_<消息uuid8>/<文件>` | `files/r12_131904_019eb513/viewer.html` | 多为重复 -> 去重后合并进 `files/<对话码>/` |
| D 现行(v2.5+) | `files/<对话码>/` + `__vN` 命名 | `files/e97526ef/viewer.html`、`viewer__v2.html` | **目标形态**,保留 |

> 对话顶层目录一直是 `<对话名>__<对话uuid前8>`,例:`claudesave1__e97526ef`。
> 目标:A、C 里的文件都并入 B/D 用的 `files/<对话码>/` 单层目录。

### 1.2 同名去重 / 版本后缀(多代混用,需要识别)
| 规律 | 来源 | 例子 | 含义 |
|------|------|------|------|
| ` (N)` 空格括号 | Chrome 下载自动加 | `cjr-writing-SKILL (2).txt`、`viewer (3).html` | 纯重复,去掉 |
| `_N` 单下划线数字 | Chrome / 某版扩展 | `preview_3.webp`、`pg-1 (1).webp` | 多为重复,靠哈希判 |
| `__N` 双下划线数字 | 老版扩展去重 | `conversation__2.json`、`prompt__3.docx` | 多为重复,靠哈希判 |
| `__<8位十六进制>` | 扩展按文件 uuid 去重 | `p_expanded__a279bc82.webp`、`page-03__2edb75e7.webp` | **版本标识**,不同 hex 多为不同文件 |
| `__vN`(v2.5+ 现行) | 扩展按内容指纹版本化 | `viewer__v2.html`、`时间管理大师_项目说明书__v2.md` | **正式版本号**,保留 |
| `<13位时间戳>_<原名>` | 上传文件带的时间戳前缀 | `1781205852796_home.txt`、`1781081059043_image.webp` | 前缀仅用于唯一化,求 baseName 时可剥 |

> `__<8位hex>` 和 `__vN` 都可能是**有意义的版本区分**,不要当纯重复无脑删;
> **一切以内容哈希为准**(第 3 节):哈希相同才是真重复。

### 1.3 图片
- 一律 `.webp`(服务器只给 webp 预览,**原始 PNG 拿不到**,别找 png)。
- 命名形态:`<13位时间戳>_image.webp`、`<32位md5>.webp`、`<语义名>.webp`、`<uuid>.webp`。

### 1.4 会话级固定文件(保留,绝不动)
- `conversation.json`、`conversation.md` -- 当前正本。
- `history/conversation_<ISO时间>.json` -- 历史快照(全部保留,查看器靠它们自动合并)。
- 根目录 `_index.json`、`_runlog.txt` -- 全局索引与运行日志(保留)。

### 1.5 垃圾 / 异常文件(移 `_trash/`)
| 模式 | 说明 |
|------|------|
| `*.crdownload`、`未确认 *.crdownload` | 下载中断的临时文件,无效 |
| `~$*.docx` / `~$*.xlsx` | Office 锁文件,无效 |
| `新建文件夹/` 等手动副本目录 | 手动整理残留,先移走再人工确认 |
| `files/` 里出现的 `<时间戳>_conversation.json`、`conversation__N.json/.md`、`conversation (N).md` | 把会话正本误当附件重复下载的产物(正本已在对话根目录)-> 移走 |

> 说明:`conversation (N).md` 这类正本副本是 v2.8 的一个回归 bug 产生的,v2.9.1 已修(正本恢复覆盖)。
> 盘上已存在的旧副本不会自动消失,由本手册移入 `_trash/`。

## 2. 目标统一结构

```
ClaudeArchive/
  _index.json
  _runlog.txt
  _trash/                                  # 所有判定为垃圾/重复的文件,移到这里(不删除)
    <原相对路径保持不变>/...
  <对话名>__<对话uuid8>/
    conversation.json
    conversation.md
    history/
      conversation_<ISO时间>.json ...
    files/
      <对话uuid前8>/                        # 唯一的产出目录(= 对话 uuid 前 8 位,如 e97526ef)
        viewer.html                        # 某文件第 1 版(原名)
        viewer__v2.html                    # 内容不同的第 2 版
        viewer__v3.html
        1781081059043_image.webp           # 图片、上传文件也都在这一层
        attachment.txt
```

规则:
- **目标只有一层产出目录** `files/<对话码>/`(`<对话码>` = 对话 uuid 前 8 位)。
- **布局 B/D**(已在 `files/<对话码>/`):保留,仅做去重 + 必要时改成 `__vN`。
- **布局 A(平铺)/ C(轮次目录)**:文件**移动并入** `files/<对话码>/`,按第 3 节去重 + 版本化;C 的空轮次目录清空后删除。
- 并入时若与目标目录已有文件**同名**:哈希相同 -> 丢弃(重复,移 `_trash/`);哈希不同 -> 存为新版本 `__v2/__v3`。

---

## 3. 去重 + 版本化(核心,必须按内容哈希)

对每个**产出文件**(非 conversation.json/.md、非 history、非 _index/_runlog)算 **SHA-256**,然后:

1. **按 baseName 分组**。baseName = 剥掉去重/版本后缀后的原始名(剥 ` (N)`、`_N`、`__N`、`__vN`、`__<8hex>`、前导 `<13位时间戳>_`;见第 7 节),保留真实扩展名。
   例:`viewer.html` / `viewer__v2.html` / `viewer (3).html` 的 baseName 都是 `viewer.html`。
2. 同一对话内,该 baseName 下**所有文件按哈希去重**:
   - 哈希相同的多份 = 真重复 -> 留一份,其余移 `_trash/`;
   - 剩下的**不同哈希** = 不同真实版本。按 mtime 从早到晚:第 1 个用 `baseName`,其余 `baseName__v2.ext`、`__v3.ext`,统一放进 `files/<对话码>/`。
3. **跨对话不合并、不去重**(不同对话各留各的)。
4. 去重**只认内容哈希**,绝不只看文件名:`p_expanded__a279bc82.webp` 与 `p_expanded__46a16423.webp` 名字像但哈希不同 = 两个文件,都保留(-> `p_expanded.webp` 与 `p_expanded__v2.webp`)。
5. **图片/上传文件同样适用**:同一张图被多次下载(不同后缀/前缀)只留一份;真不同的各留。

> 实操提示:`files/r..轮次/` 下的 `viewer.html`、各种 `.zip` 往往 N 个轮次内容完全相同,
> 去重后通常只剩 1~3 个真实版本 -- 这正是把"一堆几十 KB 重复文件夹"瘦身的关键。

---

## 4. 文件名归一(求 baseName + 重新版本化)

先把去重/版本后缀剥掉求出 baseName,再按第 3 节用哈希重新分配版本号:
- 剥掉(求 baseName):` (N)`、`_N`、`__N`、`__vN`、`__<8hex>`、前导 `<13位时间戳>_`。
- 求出每个 baseName 的"几个不同内容版本"后重命名:第 1 版 `baseName`,后续 `baseName__v2.ext`、`__v3.ext`。
- **不要保留旧的 `__<hex>` / `__N` / ` (N)` 后缀**,统一改用 `__vN`,版本号按内容首次出现的 mtime 排序。

---

## 5. 执行步骤(建议 Python:pathlib + hashlib + shutil)

> **全程先 dry-run 打印计划,确认后再真正移动。**

1. **备份**:确认已整体复制 `ClaudeArchive`。脚本开头打印警告。
2. **扫描**:遍历每个 `<对话名>__<uuid8>/`,分类每个文件:
   - 正本 / `history/` / `_index.json` / `_runlog.txt` -> 跳过不动;
   - 垃圾(1.5)-> 计划移 `_trash/`;
   - 其余产出文件 -> 记录 `(对话, 当前相对路径, 布局A/B/C/D, baseName, SHA256, mtime, 大小)`。
3. **垃圾隔离**:`shutil.move` 到 `ClaudeArchive/_trash/<原相对路径>`。
4. **按对话 + baseName 分组,去重 + 版本化**(第 3、4 节):
   - 目标目录 = 该对话 `files/<对话uuid前8>/`(没有就建);
   - 每个 baseName:同哈希留一份(余者进 `_trash/`);不同哈希按 mtime 升序命名 `baseName` / `__v2` / `__v3`,`shutil.move` 进目标目录;
   - A(平铺)、C(轮次目录)的文件都按此并入;B/D 已在目标层只做去重/重命名。
5. **清空空目录**:被搬空的 `files/r..._.../` 等删除(空目录删除安全)。
6. **产出报告**:写 `cleanup-report.csv` / `.json` 到 `ClaudeArchive/` 根,每行 `对话, 原路径, 新路径(或"->_trash"), 动作(keep/move/version/dedup/trash), 版本号, SHA256`。
7. **结束**:打印统计(对话数、移动、去重、版本、垃圾、释放空间)。**不要自动清空 `_trash/`**,人工确认后手动删。

## 6. 关键判定:目录正则

```
files 下的子目录名:
  ^r\d+_\d{6}_[0-9a-z]{8}$   # 例 r12_131904_019eb513 -> 布局 C(轮次目录,去重合并)
  ^[0-9a-f]{8}$              # 例 e97526ef           -> 布局 B/D(对话码目录 = 目标层)
  # files/ 下直接是文件      -> 布局 A(平铺,合并进对话码目录)
```

对话顶层目录名:`^(?P<name>.+)__(?P<uuid8>[0-9a-f]{8})$`(末尾 8 位十六进制 = 对话 uuid 前 8 位,即 `files/<对话码>/` 中的 `<对话码>`)。

---

## 7. 后缀剥离正则(求 baseName,第 4 节)

循环剥掉文件名末尾(扩展名之前)的下列任一后缀,直到没有;再剥一次前导时间戳:

```
\s\(\d+\)(?=\.[^.]+$)         # " (2)"
__v\d+(?=\.[^.]+$)            # "__v2"(现行版本号)
__\d+(?=\.[^.]+$)             # "__2"(老版去重)
_\d+(?=\.[^.]+$)             # "_3"
__[0-9a-f]{8}(?=\.[^.]+$)    # "__a279bc82"(老版 uuid 后缀)
^\d{13}_                      # "1781205852796_"(上传文件时间戳前缀)
```

剥离只为分组归一,**最终版本号一律用 `__vN` 重新生成**,不保留旧后缀。仅剥**紧贴扩展名之前**的后缀,别误伤正常名里的下划线数字(如 `page-1.webp` 的 `-1`、`fig2_x.webp` 中间的 `2_`)。

---

## 8. 安全红线(务必遵守)

- 绝不 `rm`/直接删除;一切"删除"= 移到 `_trash/`。
- 绝不动 `conversation.json` / `conversation.md` / `history/` / `_index.json` / `_runlog.txt`。
- 跨对话不合并、不去重。
- 改动前先 dry-run 打印完整计划并要求确认。
- `.crdownload`、`~$` 文件可直接判垃圾。

---

## 9. 给查看器的兼容性说明(无需脚本处理)

最新版查看器(`viewer.html` + `viewer.js`,需放同一目录)按"文件名(basename)"在该对话 `files/` 整个子树里查找文件,
能识别 A/B/C/D 四种历史布局,也认得 `__vN` 命名。所以**整理成什么结构都能正常打开**;
本手册的归并主要是"省空间(删掉大量重复副本)+ 眼睛舒服",不影响查看器使用。
