Claude Archive 套件 (v1.8.0)
============================

1) claude-archive/   —— 浏览器扩展
   Chrome 开 chrome://extensions(Edge: edge://extensions)→「开发者模式」
   →「加载已解压的扩展程序」→ 选 claude-archive 文件夹。正常聊天自动存到「下载/ClaudeArchive/」。
   旧对话:打开它、等加载完,点扩展弹窗「保存当前对话(全量)」。

2) 本地查看器/viewer.html  —— 仿 Claude 网页版查看器
   双击打开 →「选择存档文件夹」选 ClaudeArchive(同一对话多份 JSON / history 快照会自动合并去重)。

本次重大修复(v1.8.0):
  · 【核心】Claude 生成给你的文件现在能抓到了!之前完全漏掉——这些文件不在 JSON 的 files 字段,
    而是藏在 present_files 工具结果的 local_resource 里(带 file_path)。现在遍历这些路径,
    用 wiggle/download-file?path= 接口逐个下载。viewer 里也会在消息末尾「Claude 生成的文件」分组显示。
  · 【修复】点图片不再触发下载:改成点击开灯箱预览(图片本就在本地),下载是独立的小按钮。
    可看的文件(pdf/html/txt)点击新标签打开,zip/docx 等点下载按钮才下。
  · 历史自动合并:载入文件夹时,同一对话的多份 JSON 和 history/ 快照静默合并去重,无需手动操作。
  · 文件按路径去重,同名不同文件用 __uuid8 区分,文件夹内不冲突。

关于"遍历下载 / 接大模型":现在用确定性的路径遍历(从 present_files 提取 file_path)即可,
不需要大模型介入;接口格式严格按你给的真实链接实现,官方 claude.ai 与 claude.hk.cn 都覆盖。

全部本地运行,不上传数据。
