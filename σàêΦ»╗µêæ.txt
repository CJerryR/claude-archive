Claude Archive 套件 (v2.5.0)
============================

1) claude-archive/   —— 浏览器扩展(Chrome/Edge:开发者模式→加载已解压的扩展程序→选此文件夹)
2) 本地查看器/viewer.html —— 双击打开,「选择存档文件夹」选 ClaudeArchive
3) 工具/整理报告.html —— 扫描本地 ClaudeArchive 生成归类报告(只读)
4) 工具/整理手册_给ClaudeCode.md —— 给 Claude Code 的整理规则

本次更新(v2.5.0):彻底修正多版本文件管理逻辑
  · 不再按轮次建一堆文件夹(那会装一堆一模一样的副本)。改为:
    所有产出文件放 files/{对话码}/ 下,按【内容指纹(SHA-256)】版本化:
    同名文件只有内容真的变了才存新版本(viewer.html / viewer__v2.html / viewer__v3.html),
    内容相同的轮次直接跳过,不再产生重复副本。
  · 某文件在 11 轮出现但内容只变 3 次 → 你只会得到 3 个版本,而不是 11 个相同文件。

  关于"为什么旧版本拿不回来"(重要):
    Claude 容器里每个产出文件路径固定(如 /mnt/user-data/outputs/viewer.html),
    服务器只保留该路径的【当前最新版本】。所以历史旧版本只能在它产生的当下、
    文件还在服务器上时被抓到 —— 这正是"实时自动保存"的意义。
    事后再点保存,旧版本已被最新版覆盖,任何工具都无法找回。
    ➜ 想完整留住每个版本:保持扩展开启 + 自动保存打开,让它每轮即时存档。

  · 配合 v2.4 的"已存不重下":下载前查 chrome.downloads 记录+磁盘,已有就跳过,
    不重复下载、不弹"替换吗"。

此前已有:图片按真实格式(webp)保存、跟踪/进度/统计永久记忆、思考链工具步骤用 Claude 动作描述、
公式 KaTeX、代码块一键复制 + VSCode 高亮、Claude 风格配色、消息页脚、统计页、
多版本 JSON 自动合并去重、历史快照、查看器兼容所有历史布局。

支持 claude.ai 与 claude.hk.cn。
