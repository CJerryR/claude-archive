# ClaudeArchive 整理手册(交给 Claude Code 执行)

> 目的:把 `ClaudeArchive` 目录里**多个历史版本扩展**产生的、命名/布局各不相同的文件,
> 归类成一套**统一结构**,并安全地去重、隔离垃圾文件。
> 适用目录:`<下载目录>/ClaudeArchive`(本机示例:`C:\Users\17372\Downloads\ClaudeArchive`)。
>
> **执行前务必先整目录做一次备份(整个 ClaudeArchive 复制一份)。** 本手册所有"删除"都先移动到 `_trash/`,不直接删。

---

## 0. 一句话目标

每个对话一个顶层目录;该对话所有产出文件按**轮次**归档到 `files/<轮次>/`;
内容完全相同的重复文件只留一份;垃圾/临时文件移到 `_trash/`;历史 JSON 快照保留在 `history/`。

---

## 1. 现状:历史上出现过的全部命名 / 布局规律

### 1.1 三代文件夹布局(会同时出现在同一对话里)
| 代 | 布局 | 例子 |
|----|------|------|
| A 最早(≤v1.8) | `files/<文件>` 直接平铺 | `files/CLAUDE.txt`、`files/preview.webp` |
| B 中期(v1.9–2.1) | `files/<对话uuid前8>/<文件>` | `files/85523dd2/F_hero.webp` |
| C 现行(v2.2+) | `files/r<序号>_<时分秒>_<消息uuid8>/<文件>` | `files/r12_131904_019eb513/viewer.html` |

> 对话顶层目录一直是 `<对话名>__<对话uuid前8>`,例:`claudesave1__e97526ef`。

### 1.2 同名去重后缀(多代混用,需要识别并归并)
| 规律 | 来源 | 例子 |
|------|------|------|
| ` (N)` 空格括号 | Chrome 下载自动加 | `cjr-writing-SKILL (2).txt` |
| `_N` 单下划线数字 | Chrome / 某版扩展 | `preview_3.webp`、`e677c037_1.webp`、`p_expanded_20.webp` |
| `__N` 双下划线数字 | 老版扩展去重 | `conversation__2.json`、`attachment__3.txt`、`prompt__3.docx` |
| `__<8位十六进制>` | 扩展按文件 uuid 去重 | `p_expanded__a279bc82.webp`、`page-03__2edb75e7.webp`、`preview__0b8a48a2.webp` |

> 注意:`__<8位hex>` 后缀是**有意义的版本区分**(不同 uuid = 不同文件),不要当成纯重复无脑删;
> 要靠**内容哈希**判断是否真重复(见第 3 节)。

### 1.3 图片
- 一律是 `.webp`(服务器只提供 webp 预览,**原始 PNG 无法获得**,别去找 png)。
- 命名形态:`<13位时间戳>_image.webp`、`<32位md5哈希>.webp`、`<语义名>.webp`、`<uuid>.webp`。

### 1.4 会话级固定文件(保留,不动)
- `conversation.json`、`conversation.md` —— 当前版本的正本。
- `history/conversation_<ISO时间>.json` —— 历史快照(全部保留,查看器用它们自动合并)。
- 根目录 `_index.json` —— 全局索引(保留)。

### 1.5 垃圾 / 异常文件(移到 `_trash/`,不要算进正式归档)
| 模式 | 说明 |
|------|------|
| `未确认 *.crdownload`、`*.crdownload` | 下载中断的临时文件,无效 |
| `~$*.docx`、`~$*.xlsx` | Office 打开时的锁文件,无效 |
| `新建文件夹/`(及任何手动建的副本目录) | 你手动整理时的残留副本,先移走再人工确认 |
| 形如 `<时间戳>_conversation.json`、`conversation__N.json/.md` 出现在 `files/` 里 | 是把会话正本误当附件重复下载的产物,正本已在对话根目录,这些可移走 |

## 2. 目标统一结构

```
ClaudeArchive/
  _index.json
  _trash/                                  # 所有被判定为垃圾/重复的文件,移动到这里(不删除)
    <原相对路径保持不变>/...
  <对话名>__<对话uuid8>/
    conversation.json
    conversation.md
    history/
      conversation_<ISO时间>.json ...
    files/
      r<序号>_<时分秒>_<消息uuid8>/         # 每一轮 Claude 产出 → 一个文件夹
        <产出文件>...
      _legacy/                             # 无法判定属于哪一轮的老文件(来自布局 A/B)归到这里
        <产出文件>...
```

规则:
- **布局 C(已是轮次目录)**:原样保留。
- **布局 B(`files/<对话码>/`)与布局 A(`files/` 平铺)**:文件移动到 `files/_legacy/`
  (因为老布局没有记录是哪一轮产出的,无法可靠还原轮次;统一进 `_legacy` 便于查找,且不与轮次目录混淆)。
- 若 `_legacy/` 内与某个轮次目录里**内容完全相同**(哈希一致)的文件,则 `_legacy/` 里那份是重复 → 移到 `_trash/`。

> 备选(若你更想"全部铺平"):也可以把所有产出文件直接放到 `files/_all/`,按 `第3节` 去重,文件名用 `<原名>__<内容哈希前8>.<扩展名>` 保证不撞名。手册默认用上面的 `_legacy` 方案,二选一即可。

---

## 3. 去重规则(核心,必须按内容哈希)

1. 对每个**产出文件**(非 conversation.json/.md、非 history、非 _index.json)计算 **SHA-256**。
2. 同一对话目录内,**哈希相同**的文件即真重复:
   - 保留路径"最规范"的一份,优先级:`files/r..轮次/` > `files/_legacy/` > 其它;
   - 同优先级保留**文件名最短/无去重后缀**的那个(如 `viewer.html` 优于 `viewer__2.html`);
   - 其余移到 `_trash/`(保持原相对路径)。
3. **跨对话**的相同哈希**不要合并**(不同对话本就该各留各的)。
4. 去重只看内容哈希,**不要只看文件名**——`p_expanded__a279bc82.webp` 与 `p_expanded__46a16423.webp`
   文件名像但哈希不同,是两个不同文件,都要保留。

---

## 4. 文件名归一(去掉冗余去重后缀)

把下列后缀**仅在不产生新冲突时**去掉,让文件名干净:
- ` (N)`、`_N`(纯数字)、`__N`(纯数字)→ 去掉。
- `__<8位hex>` → **保留**(这是版本标识)。

去后缀后若与同目录已有文件**同名但内容不同** → 不要覆盖,改用 `<原名>__<内容哈希前8>.<扩展名>`。
去后缀后若同名且内容相同 → 属于重复,按第 3 节处理。

---

## 5. 执行步骤(建议 Claude Code 按此顺序写脚本,Python 优先)

> 语言随意,Python `pathlib`+`hashlib`+`shutil` 最顺手。**全程 dry-run 先打印计划,确认后再真正移动。**

1. **备份**:确认已整体复制 `ClaudeArchive`。脚本开头打印警告。
2. **扫描**:遍历 `ClaudeArchive/*/`(每个对话目录),分类每个文件:
   - 是否 conversation 正本 / history / _index → 跳过不动;
   - 是否垃圾(第 1.5 节)→ 计划移入 `_trash/`;
   - 否则是产出文件 → 记录 `(对话, 当前相对路径, 布局A/B/C, 文件名, SHA256, 大小)`。
3. **垃圾隔离**:把垃圾文件 `shutil.move` 到 `ClaudeArchive/_trash/<原相对路径>`。
4. **布局归并**:
   - C 类原样;
   - A/B 类移动到该对话的 `files/_legacy/`(保留文件名;若 `_legacy` 内撞名见第 4 节)。
5. **去重**:在每个对话内按 SHA-256 分组,按第 3 节保留一份、其余移入 `_trash/`。
6. **文件名归一**:按第 4 节清理后缀。
7. **产出报告**:写一个 `整理报告.csv`/`.json` 到 `ClaudeArchive/` 根:
   每行 `对话, 原路径, 新路径(或"移入_trash"), 动作(keep/move/dedup/trash), SHA256`。
8. **结束**:打印统计(处理对话数、移动数、去重数、垃圾数、释放空间)。**不要自动清空 `_trash/`**,留给人工最终确认后手动删。

---

## 6. 关键判定:轮次目录名正则

```
^r\d+_\d{6}_[0-9a-f]{8}$        # 例 r12_131904_019eb513  → 布局 C
^[0-9a-f]{8}$                   # 例 85523dd2            → 布局 B(对话码目录)
# 都不匹配且直接在 files/ 下     → 布局 A(平铺)
```

对话顶层目录名:`^(?P<name>.+)__(?P<uuid8>[0-9a-f]{8})$`(末尾 8 位十六进制是对话 uuid 前 8 位)。

---

## 7. 去重后缀识别正则(用于第 4 节)

```
去掉(仅无冲突时): \s\(\d+\)(?=\.[^.]+$)      # " (2)"
                   _(\d+)(?=\.[^.]+$)         # "_3"
                   __(\d+)(?=\.[^.]+$)        # "__2"
保留:             __[0-9a-f]{8}(?=\.[^.]+$)  # "__a279bc82"(版本标识)
```

---

## 8. 安全红线(务必遵守)

- 绝不 `rm`/直接删除;一切"删除"= 移动到 `_trash/`。
- 绝不动 `conversation.json` / `conversation.md` / `history/` / `_index.json`。
- 跨对话不合并、不去重。
- 改动前先 dry-run 打印完整计划并要求确认。
- `.crdownload`、`~$` 文件可直接判为垃圾(它们对内容无意义)。

---

## 9. 给查看器的兼容性说明(无需脚本处理)

最新版查看器(viewer.html)已能识别 A/B/C 三种布局,以及 `files/_legacy/`、`files/_all/`:
它按"文件名(basename)"在该对话的 `files/` 全树里查找,所以**整理成什么结构都能正常打开**。
因此本手册的归并主要是"让你眼睛舒服 + 省空间",不影响查看器使用。
